# v0.10.0

Released: July 2026

## Highlights

This release focuses on **security hardening**, **operational resilience**, and **test coverage**.
All new code follows secure coding best practices with comprehensive input validation, secret
redaction, and defense-in-depth patterns.

## New Features

### Structured Logging (`cloud/logging_config.py`)

- JSON and human-readable log output formats
- Automatic redaction of secrets (AWS keys, Bearer tokens, passwords) in all log messages
- Log rotation with configurable file size and backup count
- CLI flags: `--log-level`, `--log-json`, `--log-file`
- Environment variables: `PAGERWESI_LOG_LEVEL`, `PAGERWESI_LOG_JSON`, `PAGERWESI_LOG_FILE`

### Rate Limiting & Retry (`cloud/rate_limiter.py`)

- Token-bucket rate limiter for cloud API calls
- Exponential backoff retry with configurable max retries and delay
- Automatic detection of retriable errors (throttling, timeouts, 5xx)
- Thread-safe implementation for concurrent operations

### Input Validation (`cloud/input_validator.py`)

- Path traversal protection (null byte injection, symlink checks)
- Hostname and IP address validation
- Port range validation (1–65535)
- Endpoint format validation with count limits
- Webhook URL validation with SSRF protection (rejects private/loopback IPs, enforces HTTPS)
- Control ID format validation
- AWS profile and region format validation
- Evidence sanitization (control character removal, length limits)

### Concurrent Provider Execution (`cloud/concurrent_runner.py`)

- Thread-safe parallel provider auditing for the `all` command
- Error isolation (one provider failure does not affect others)
- Duration tracking per provider
- Configurable worker count (1–32)
- Regional concurrent execution support

### Enhanced Secrets Scanner

- 12 detection patterns (up from 3):
  AWS Access/Secret keys, private keys, passwords, GitHub/GitLab tokens, Slack tokens,
  JWT tokens, database URLs, Azure connection strings, GCP service account keys, Bearer tokens
- Configurable allowlist for false positive suppression
- File size limit (1 MiB) to prevent memory exhaustion
- Per-pattern severity levels (CRITICAL, HIGH, MEDIUM)

### Agent Mode Improvements

- State history rotation (last 30 states preserved for trend analysis)
- Resolved findings tracking
- Structured logging throughout
- Minimum interval enforcement (30 seconds)
- Graceful degradation on partial provider failures

## Security Improvements

- SSRF protection on all webhook notifications
- Secret auto-redaction in all logging output
- Input validation on all CLI-provided data
- Payload size limits (64 KiB for webhooks, 50 MiB for manifests)
- SSL context enforcement for webhook HTTPS connections
- HTML entity escaping in dashboard reports (XSS prevention)
- `from None` / `from exc` patterns for proper exception chaining

## Breaking Changes

None. All existing CLI interfaces and output formats remain unchanged.

## Upgrade Notes

- New CLI flags (`--log-level`, `--log-json`, `--log-file`) are optional with sensible defaults
- The `all` command now runs providers concurrently (previously sequential)
- Webhook notifications now require HTTPS URLs (HTTP was never recommended)
- Agent state file format is unchanged; history directory (`.pagerwesi-history/`) is new
