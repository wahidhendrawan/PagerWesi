# PagerWesi v0.8.0

Released on 2026-06-18.

## Highlights

- Added a reusable JSON Schema for findings at
  [finding.schema.json](../finding.schema.json).
- Added regression coverage that validates rendered JSON findings against the schema.
- Added `--watch-providers` for agent mode and preserved the agent exit code in the CLI.
- Agent provider runtime failures now emit `AGENT-PROVIDER-001` error findings instead of being
  silently ignored.
- Webhook, dashboard, and compliance exports now normalize both live `Finding` objects and
  serialized finding dictionaries.
- Completed control metadata for Docker, secrets, Terraform plan, and network/TLS scanners so
  SARIF, compliance mappings, and generated docs stay aligned.
- Clarified that `pagerwesi all` is the core cloud provider set: AWS, Azure, GCP, and
  Kubernetes.
- Pinned the LocalStack integration image to avoid upstream `latest` license-token drift.
- Release workflow now installs the Kubernetes extra before running the release quality gate.
- README, Quickstart, and GitHub Pages were updated for local scanners, schema usage, dashboard
  generation, compliance evidence, and agent mode.

## Container

```bash
docker pull ghcr.io/wahidhendrawan/pagerwesi:v0.8.0
docker run --rm -e AWS_ACCESS_KEY_ID -e AWS_SECRET_ACCESS_KEY \
  ghcr.io/wahidhendrawan/pagerwesi:v0.8.0 aws --format json --output /dev/stdout
```

